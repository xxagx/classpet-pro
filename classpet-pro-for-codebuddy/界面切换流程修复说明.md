# 🎯 界面切换流程修复说明

## 问题描述

之前存在的问题：
- 点击"开始旅程"按钮后，登录界面和宠物界面同时显示
- 两个页面上下叠加，用户体验混乱
- 没有正确的界面切换流程控制

## 修复方案

### 1. 初始状态优化 ✅

**修改文件**: `js/mobile.js` - `initApp()` 函数

**修复内容**:
```javascript
function initApp() {
    // ... 其他初始化代码 ...
    
    // 初始化时不显示任何页面，等待用户点击"开始旅程"按钮
    // 确保 authPage 和 petPage 都没有 active 类
    authPage.classList.remove('active');
    petPage.classList.remove('active');
}
```

**效果**:
- ✅ 页面加载时只显示欢迎弹窗
- ✅ 登录界面和宠物界面都隐藏
- ✅ 避免页面叠加显示

### 2. "开始旅程"按钮点击后的流程 ✅

**修改文件**: `js/mobile.js` - `initEventListeners()` 函数

**修复内容**:

btnWelcome.addEventListener('click', () => {

### 4. 登录成功后的界面切换 ✅

**修改文件**: `js/mobile.js` - `handleLoginSuccess()` 函数

**修复内容**:
```javascript
function handleLoginSuccess(user) {
    showLoading('正在登录...');
    
    setTimeout(() => {
        const studentId = Number(user.studentId);
        currentStudent = dataManager.getStudentById(studentId);
        
        if (currentStudent) {
            showPetPage();  // 切换到宠物页面
            hideLoading();
            showAlert(`欢迎回来，${currentStudent.name}！`, '登录成功');
        } else {
            hideLoading();
            showAlert('学生信息不存在，请联系老师', '错误');
        }
    }, 500);
}
```

**界面切换**:
```javascript
function showPetPage() {
    authPage.classList.remove('active');  // 隐藏登录页
    petPage.classList.add('active');      // 显示宠物页
    updatePetDisplay();                   // 更新宠物显示
    startStatusUpdates();                 // 启动状态更新
}
```

### 5. 登录失败处理 ✅

**修改文件**: `js/auth-manager.js` - `showLoginError()` 函数

**修复内容**:
```javascript
showLoginError(message) {
    // 移除旧的错误提示
    const oldError = document.querySelector('.login-error');
    if (oldError) {
        oldError.remove();
    }
    
    // 创建新的错误提示
    const errorDiv = document.createElement('div');
    errorDiv.className = 'login-error';
    errorDiv.style.cssText = `
        background: rgba(244, 67, 54, 0.9);
        color: white;
        padding: 12px;
        border-radius: 8px;
        margin-bottom: 15px;
        font-size: 14px;
        text-align: center;
        animation: shake 0.3s ease-in-out;
    `;
    errorDiv.textContent = message;
    
    const form = document.getElementById('studentLoginForm');
    if (form) {
        form.insertBefore(errorDiv, form.querySelector('button'));
    }

    // 3 秒后自动移除
    setTimeout(() => {
        if (errorDiv.parentNode) {
            errorDiv.remove();
        }
    }, 3000);
}
```

**错误提示特性**:
- ✅ 红色背景，醒目提示
- ✅ Shake 动画，增强视觉效果
- ✅ 自动消失，3 秒后移除
- ✅ 插入到按钮上方，位置明显

### 6. 状态管理 ✅

**登录状态保存**:
- 使用 `localStorage` 保存登录状态
- 记录当前用户信息（用户名、角色、学生 ID）
- 支持"记住我"功能

**状态检查**:
```javascript
function checkLoginStatus() {
    if (authManager.isLoggedIn()) {
        const user = authManager.getCurrentUser();
        const studentId = Number(user.studentId);
        currentStudent = dataManager.getStudentById(studentId);
        if (currentStudent) {
            // 已登录，直接显示宠物页面
            authPage.classList.remove('active');
            petPage.classList.add('active');
            updatePetDisplay();
            startStatusUpdates();
            return true;
        }
    }
    return false;
}
```

## 技术要点

### 1. 页面显示控制

**CSS 控制**:
```css
.page {
    display: none;      /* 默认隐藏 */
    opacity: 0;         /* 透明度 0 */
    transition: opacity 0.3s ease-in-out;  /* 过渡动画 */
}

.page.active {
    display: block;     /* 激活时显示 */
    opacity: 1;         /* 透明度 1 */
}
```

**JavaScript 控制**:
- 添加 active 类：`classList.add('active')`
- 移除 active 类：`classList.remove('active')`

### 2. 状态管理

**登录状态**:
- 保存：`authManager.login()` 写入 localStorage
- 检查：`authManager.isLoggedIn()` 读取 localStorage
- 清除：`authManager.logout()` 删除 localStorage

**当前用户**:
- 使用 `currentStudent` 变量保存
- 登录成功后赋值
- 退出时清空

### 3. 事件处理

**自定义事件**:
```javascript
// 触发登录成功事件
const event = new CustomEvent('loginSuccess', { detail: result.user });
container.dispatchEvent(event);

// 监听登录成功事件
authContainer.addEventListener('loginSuccess', (e) => {
    const user = e.detail;
    handleLoginSuccess(user);
});
```

### 4. 错误处理

**错误提示**:
- 动态创建错误提示元素
- 插入到表单中显眼位置
- 使用 shake 动画增强效果
- 3 秒后自动移除

**错误类型**:
- 账号不存在
- 密码错误
- 账号已被禁用
- 学生信息不存在

## 常见问题

### Q1: 为什么点击"开始旅程"后没有反应？
**A**: 检查浏览器控制台是否有错误，确认 JavaScript 已启用。

### Q2: 登录后为什么看不到宠物？
**A**: 检查账号是否正确，确认学生信息存在于数据库中。

### Q3: 错误提示为什么不显示？
**A**: 检查表单是否正确初始化，确认 `studentLoginForm` 元素存在。

### Q4: 界面切换有闪烁怎么办？
**A**: 确保 CSS transition 正确应用，检查是否有其他样式覆盖。
